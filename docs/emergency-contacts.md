## 📞 연락 우선순위

### Level 1: 일반 이슈 (0-15분)

**대응자:** 담당 엔지니어

**연락 방법:**
1. Slack 채널: #bos-ai-infra
2. 이메일: team@example.com

**대응 시간:** 15분 이내

**에스컬레이션 조건:**
- 15분 내 해결 불가
- 서비스 영향도 증가
- 추가 리소스 필요

### Level 2: 중요 이슈 (15-30분)

**대응자:** 팀 리더

**연락 방법:**
1. Slack 채널: #bos-ai-infra-urgent
2. 전화: 팀 리더 직통
3. SMS: 긴급 문자

**대응 시간:** 30분 이내

**에스컬레이션 조건:**
- 30분 내 해결 불가
- 서비스 중단 발생
- 롤백 필요

### Level 3: 긴급 이슈 (30분+)

**대응자:** 프로젝트 리더

**연락 방법:**
1. 전화: 프로젝트 리더 직통 (최우선)
2. Slack: @[project-leader] 멘션
3. SMS: 긴급 문자
4. 이메일: 긴급 이메일

**대응 시간:** 즉시

**에스컬레이션 조건:**
- 서비스 완전 중단
- 데이터 손실 위험
- 보안 침해 의심
- 경영진 보고 필요

## 🚨 긴급 상황별 연락 절차

### 1. 서비스 중단

**1차 연락:** 운영 담당 → 인프라 담당  
**2차 연락:** 팀 리더  
**3차 연락:** 프로젝트 리더

**연락 채널:**
- Slack: #bos-ai-incident
- 전화: 순차적 연락
- PagerDuty: 자동 알림

### 2. 보안 이슈

**1차 연락:** 보안 담당 (즉시)  
**2차 연락:** 프로젝트 리더 (즉시)  
**3차 연락:** CISO (필요시)

**연락 채널:**
- 전화: 보안 담당 직통 (최우선)
- Slack: #security-incident (비공개)
- 이메일: security@example.com

### 3. 데이터 손실

**1차 연락:** 개발 담당 → 인프라 담당  
**2차 연락:** 프로젝트 리더 (즉시)  
**3차 연락:** DPO (필요시)

**연락 채널:**
- 전화: 순차적 연락
- Slack: #bos-ai-data-incident
- 이메일: data-protection@example.com

### 4. 비용 초과

**1차 연락:** 인프라 담당  
**2차 연락:** 프로젝트 리더  
**3차 연락:** 재무 담당

**연락 채널:**
- Slack: #bos-ai-cost-alert
- 이메일: finance@example.com

      "response_time_seconds": 30
    },
    {
      "name": "인프라 담당",
      "method": "phone",
      "time": "2026-02-22T10:31:00Z",
      "response_time_seconds": 60
    }
  ],
  "resolution_time_minutes": 15,
  "actions_taken": ["롤백 실행", "서비스 복구"]

```bash
# 최근 연락 이력
aws logs filter-log-events \
  --log-group-name /aws/bos-ai/emergency-contacts \
  --start-time $(date -u -d '30 days ago' +%s)000 \
  --filter-pattern '{ $.severity = "critical" }'
```
