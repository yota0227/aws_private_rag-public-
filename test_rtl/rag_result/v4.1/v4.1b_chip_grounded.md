## 1. Overview

### 1.1 Chip Purpose

Trinity N1B0 is a multi-tile AI accelerator SoC built on a 2D mesh grid. The chip integrates Tensix compute tiles (each containing FPU and SFPU datapaths), a RISC-V Overlay CPU subsystem, NoC-based interconnect with AXI bridging, an SMN security/memory network, and an EDC serial diagnostic ring.

The following blocks have HDD sections or structural claims confirmed in the knowledge base for pipeline `tt_20260221`:

## 9. EDC (Error Detection and Correction)

### 9.1 KB-Confirmed Interface

*Source: module_parse results for `tt_edc_pkg.sv`, pipeline tt_20260221*

The EDC package defines a SystemVerilog interface with the following modports:

| Modport | Signals (Direction from modport's perspective) |
|---|---|
| `ingress` | `input req_tgl`, `output ack_tgl`, `input cor_err`, `output err_inj_vec` |
| `egress` | `output req_tgl`, `input ack_tgl`, `output cor_err`, `input err_inj_vec` |
| `edc_node` | `input req_tgl`, `output ack_tgl`, `output req_tgl`, `input ack_tgl`, `input cor_err`, `output err_inj_vec`, `output cor_err`, `input err_inj_vec` |
| `sram` | `input err_inj_vec`, `output cor_err` |

### 9.2 Signal Descriptions (KB-Confirmed)

| Signal | Width | Description |
|---|---|---|
| `req_tgl` | 1 | Request toggle — level transition indicates new data beat |
| `ack_tgl` | 1 | Acknowledge toggle — receiver confirms data consumption |
| `cor_err` | varies | Correctable error flag from SRAM ECC logic |
| `err_inj_vec` | varies | Error injection vector — flip bits in SRAM read data for DFT |

### 9.3 Serial Bus Protocol

**KB-confirmed:** The EDC uses a **toggle-handshake** protocol (`req_tgl`/`ack_tgl`), which is inherently asynchronous — no dedicated EDC clock is required on the serial link.

### 10.2 Interrupt CDC (KB-Confirmed)

- **Module:** `TTTrinityConfig_IntSyncAsyncCrossingSink_n1x1`
- **Function:** Asynchronous-to-synchronous interrupt crossing sink (1-bit wide, 1×1)
- **Synchronizer:** `TTTrinityConfig_SynchronizerShiftReg_w1_d3` — 3-stage shift register (w=1 bit, d=3 stages)
- **Purpose:** Safely transfers interrupt signals across clock domain boundaries within the Overlay

### 10.3 Chipyard Configuration

- **Config name:** `TTTrinityConfig` (encoded in module names)
- **Generated by:** Chipyard framework (indicated by `chipyard.harness.TestHarness.TTTrinityConfig` path)
