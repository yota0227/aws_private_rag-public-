## 1. Overview

Dispatch EngineмқҖ Trinity N1B0 нғҖмқјм—җм„ң **нҳёмҠӨнҠё CPU лҳҗлҠ” мқём ‘ нғҖмқјлЎңл¶Җн„° мҲҳмӢ лҗң лӘ…л №/лҚ°мқҙн„°лҘј Tensix AI мҪ”м–ҙлЎң м „лӢ¬(dispatch)н•ҳлҠ” м„ңлёҢмӢңмҠӨн…ң**мқҙлӢӨ.

NoC вҶ” AXI лҸ„л©”мқё к°„мқҳ лқјмҡ°нҢ…кіј н”„лЎңнҶ мҪң ліҖнҷҳмқ„ нҸ¬н•Ён•ҳл©°, нғҖмқј лӮҙм—җм„ң **лӘ…л № л¶„л°°(command dispatch)** л°Ҹ **лҚ°мқҙн„° лқјмҡ°нҢ…(data routing)**мқ„ лӘЁл‘җ лӢҙлӢ№н•ңлӢӨ.

### н•өмӢ¬ м„Өкі„ лӘ©н‘ң

| лӘ©н‘ң | кө¬нҳ„ |
|------|------|
| **м Җм§Җм—° лӘ…л № м „лӢ¬** | NoC нҢЁнӮ· мҲҳмӢ  вҶ’ Tensix мҪ”м–ҙ лӘ…л № нҒҗк№Ңм§Җ мөңмҶҢ нҢҢмқҙн”„лқјмқё лӢЁкі„ |
| **л°©н–Ҙлі„ нҠёлһҳн”Ҫ л¶„мӮ°** | East/West л“ұ л°©н–Ҙлі„ лҸ…лҰҪ л””мҠӨнҢЁм№ҳ мң лӢӣ мқёмҠӨн„ҙмҠӨнҷ” |
| **NoCвҶ”AXI н”„лЎңнҶ мҪң лёҢлҰҝм§Җ** | `trinity_noc2axi_router_ne_opt_FBLC`лҘј нҶөн•ң лҸ„л©”мқё к°„ ліҖнҷҳ |
| **кіөм •н•ң мӨ‘мһ¬** | Round-Robin ArbiterлҘј нҶөн•ң лӢӨмӨ‘ мҶҢмҠӨ лӘ…л № мҠӨмјҖмӨ„л§Ғ |

## 10. Summary

Dispatch EngineмқҖ Trinity N1B0 нғҖмқјм—җм„ң **лӘ…л № мҲҳмӢ  вҶ’ л””мҪ”л”© вҶ’ Tensix мҪ”м–ҙ м „лӢ¬**мқҳ м „мІҙ нҢҢмқҙн”„лқјмқёмқ„ лӢҙлӢ№н•ҳлҠ” м„ңлёҢмӢңмҠӨн…ңмңјлЎң, л‘җ к°Җм§Җ н•өмӢ¬ лё”лЎқмңјлЎң кө¬м„ұлҗңлӢӨ:

1. **`trinity_noc2axi_router_ne_opt_FBLC`** вҖ” NoC нҢЁнӮ·мқ„ AXI нҠёлһңмһӯм…ҳмңјлЎң ліҖнҷҳн•ҳлҠ” лқјмҡ°н„°. NE мҝјл“ңлҹ°нҠё мөңм Ғнҷ”, FBLC 4л°©н–Ҙ нҸ¬нҠё, CDC FIFOлҘј нҸ¬н•Ё. 76кұҙмқҳ Dispatch нҶ н”Ҫ лҚ°мқҙн„°м—җм„ң к°ҖмһҘ л№ҲлІҲн•ҳкІҢ л“ұмһҘн•ҳлҠ” лӘЁл“ҲлЎң, Dispatch кІҪлЎңмқҳ н”„лЎ нҠём—”л“ң м—ӯн• мқ„ мҲҳн–үн•ңлӢӨ.

2. **`tt_dispatch_top_*`** вҖ” л°©н–Ҙлі„(East/West/N/S)лЎң мқёмҠӨн„ҙмҠӨнҷ”лҗң л””мҠӨнҢЁм№ҳ мң лӢӣ. AXI нҠёлһңмһӯм…ҳмқ„ лӘ…л №мңјлЎң л””мҪ”л”©н•ҳм—¬ Tensix мҪ”м–ҙмқҳ лӘ…л № нҒҗм—җ м Ғмһ¬н•ҳл©°, Credit кё°л°ҳ Flow ControlлЎң л°°м••мқ„ кҙҖлҰ¬н•ңлӢӨ.

### лҚ°мқҙн„° кІҪлЎң мҡ”м•Ҫ

```
NoC Packet вҶ’ Router (FBLC) вҶ’ Protocol Convert вҶ’ Dispatch Top вҶ’ Command Queue вҶ’ Tensix Execute
```

> рҹ“Ң **Note:** мқҙ HDDлҠ” 76кұҙмқҳ Dispatch нҶ н”Ҫ лҚ°мқҙн„° мӨ‘ л°ҳнҷҳлҗң мғҒмң„ 5кұҙ(лӘЁл‘җ hdd_section мң нҳ•)кіј мқҙм „ кІҖмғүм—җм„ң нҷ•мқёлҗң Trinity нғҖмқј кө¬мЎ° м •ліҙлҘј кё°л°ҳмңјлЎң мһ‘м„ұлҗҳм—ҲмҠөлӢҲлӢӨ. Claim мң нҳ•(behavioral, structural, connectivity) лҚ°мқҙн„°к°Җ лӮҳлЁём§Җ 71кұҙм—җ нҸ¬н•Ёлҗҳм–ҙ мһҲмқ„ к°ҖлҠҘм„ұмқҙ лҶ’мңјл©°, м¶”к°Җ кІҖмғү мӢң Dispatch лӮҙл¶Җ л Ҳм§ҖмҠӨн„° л§ө, мғҒм„ё FSM, м—җлҹ¬ н•ёл“Өл§Ғ лЎңм§Ғ л“ұмқҳ ліҙмҷ„мқҙ к°ҖлҠҘн•©лӢҲлӢӨ.
