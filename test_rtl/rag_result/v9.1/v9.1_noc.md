## 1. Overview

The Network-on-Chip (NoC) fabric provides the primary interconnect for the Trinity N1B0 NPU. It implements a 4×5 2D mesh topology with support for deterministic, adaptive, and dynamic routing algorithms. The fabric includes ECC protection, clock domain crossing infrastructure, and configurable timeout mechanisms.

Key parameters:
- Grid: 4×5 (SizeX=4, SizeY=5, NumNodes=20)
- Axes: 2 (NumAxes=2)
- Directions per axis: 2 (NumDirections=2)
- Dynamic routing: Enabled (EnableDynamicRouting=1'b1)
- AXI bridges: 4 (NumNoc2Axi=4)

Note: `trinity_router` is NOT instantiated in N1B0 (EMPTY by design).

## 2. Routing Algorithms

| Algorithm | Type | Description |
|-----------|------|-------------|
| DIM_ORDER | Deterministic | XY dimension-ordered routing — packets route in X first, then Y |
| TENDRIL | Adaptive | Congestion-aware routing — selects path based on link utilization |
| DYNAMIC | Dynamic | Runtime-selectable routing — enabled via EnableDynamicRouting=1'b1 |

### 2.1 Routing Configuration

- Default: DIM_ORDER (deadlock-free, minimal path)
- EnableDynamicRouting parameter enables runtime algorithm selection
- TENDRIL provides load balancing across alternative paths

## 6. Arbitration Architecture

### 6.1 Tree Structure

```
noc_arbiter_tree
├── Level 0: Per-port request inputs
├── Level 1: Pairwise priority comparison
├── Level 2: Winner selection
└── Output: Grant + selected data
```

### 6.2 Arbitration Points

- Each router node has multiple arbiter instances
- One per output port (N, S, E, W, Local)
- Priority-based with fairness mechanism

## 8. Grid Topology

### 8.1 Mesh Layout (4×5)

```
        Col 0    Col 1    Col 2    Col 3
Row 4  [Node]   [Node]   [Node]   [Node]   ← North edge (isNorthEdge)
Row 3  [Node]   [Node]   [Node]   [Node]
Row 2  [Node]   [Node]   [Node]   [Node]
Row 1  [Node]   [Node]   [Node]   [Node]
Row 0  [Node]   [Node]   [Node]   [Node]
                                     ↑
                              East edge (isEastEdge)
```

### 8.2 Edge Detection Functions

| Function | Condition | Usage |
|----------|-----------|-------|
| isEastEdge(x) | x == SizeX - 1 (x == 3) | Boundary routing decisions |
| isNorthEdge(y) | y == SizeY - 1 (y == 4) | Boundary routing decisions |

### 8.3 Inter-Column Repeaters

- `tt_noc_repeaters_cardinal` placed between columns
- Regenerates signals for timing closure across wide mesh
- Cardinal directions: connects N/S/E/W links

## 9. Harvest Support

### 9.1 tt_harvest_robust_sync

- **Function:** Robust multi-replication synchronizer for harvest configuration
- **Purpose:** Safely propagates tile enable/disable signals across clock domains
- **Technique:** Multi-stage replication for metastability protection

### 9.2 Integration with EDC

- EDC ring uses loopback connectors to bypass harvested tiles
- NoC routing tables updated to exclude harvested nodes
- Harvest configuration synchronized via tt_harvest_robust_sync
