## 1. Overview

The EDC (Error Detection and Correction) subsystem provides error management for the Trinity N1B0 NPU. It uses a ring topology connecting all nodes in the 4×5 mesh grid, with APB4 register access and 5 interrupt lines for error reporting.

Note: `trinity_router` is NOT instantiated in N1B0 (EMPTY by design).

## 3. Module Details

### 3.1 tt_edc1_intf_connector

- **Function:** Ring node connector
- **Ports:** None (passthrough connector)
- **Instances in trinity.sv:**
  - `edc_direct_conn_nodes` — Direct connection path
  - `edc_loopback_conn_nodes` — Loopback connection path

### 3.2 tt_edc1_serial_bus_repeater

- **Function:** Serial bus repeater for ring signal regeneration
- **Ports:**
  - `i_clk` — Clock input
  - `i_reset_n` — Active-low reset
- **Features:** Contains SystemVerilog property assertion for protocol checking

### 3.3 tt_edc1_biu_soc_apb4_wrap

- **Function:** APB4 slave interface wrapper for SoC integration
- **Inner module:** `edc1_biu_soc_apb4_inner`
- **APB4 Input Signals:**
  - `psel` — Peripheral select
  - `penable` — Enable phase
  - `pwrite` — Write/read direction
  - `pprot[2:0]` — Protection type
  - `paddr[5:0]` — Address (6-bit, 64 registers)
  - `pwdata[31:0]` — Write data
  - `pstrb[3:0]` — Write strobe
- **APB4 Output Signals:**
  - `pready` — Transfer ready
  - `prdata[31:0]` — Read data
  - `pslverr` — Slave error
- **IRQ Outputs (5 lines):**
  - `o_edc_fatal_err_irq` — Fatal error interrupt
  - `o_edc_crit_err_irq` — Critical error interrupt
  - `o_edc_cor_err_irq` — Correctable error interrupt
  - `o_edc_pkt_sent_irq` — Packet sent interrupt
  - `o_edc_pkt_rcvd_irq` — Packet received interrupt

### 3.4 tt_edc1_noc_sec_block_reg

- **Function:** Security configuration registers for NoC access control
- **Inner module:** `edc1_noc_sec_block_reg_inner`
- **Purpose:** Configures per-node security policies for EDC ring access

### 4.3 Harvest Support

- Mux/demux at each node for harvest bypass
- Loopback nodes enable ring continuity when tiles are harvested
- Maintains ring integrity regardless of active tile count
