## 1. Overview

The Overlay subsystem in Trinity N1B0 provides the RISC-V based control and configuration infrastructure. It includes register file modules for Tensix tiles, dispatch engines, and cluster control, enabling software-programmable configuration of the compute fabric.

Key parameters:
- NumDmComplexes = 14 (DM complex count)
- DMCoresPerCluster = 8 (RISC-V cores per cluster)
- Total DM cores: 14 × 8 = 112

Note: `trinity_router` is NOT instantiated in N1B0 (EMPTY by design).

## 6. Integration with Compute Fabric

### 6.1 Tensix Control

- tt_fds_tensixneo_reg controls all 12 Tensix tiles
- Per-tile configuration via register address decoding
- Supports runtime reconfiguration without full reset

### 6.2 Dispatch Control

- tt_fds_dispatch_reg manages both East and West dispatch engines
- Coordinates work distribution across the 4×5 mesh
- Enables asymmetric workload assignment

### 6.3 L1/Cluster Control

- tt_cluster_ctrl_t6_l1_csr_reg manages cluster-level resources
- 97 output signals provide fine-grained control
- Includes L1 cache partitioning and allocation policies

- ISO_EN[11:0] controlled via overlay register writes
