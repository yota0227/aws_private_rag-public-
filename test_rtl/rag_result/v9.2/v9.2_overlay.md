## 1. Overview

The Overlay subsystem is the RISC-V CPU cluster and glue logic within Trinity N1B0. It provides programmable control, DMA, and system management functions for the compute tiles.

Key characteristics:
- CPU cluster with 8 RISC-V cores per DM complex (DMCoresPerCluster = 8)
- 14 DM complexes total (NumDmComplexes = 14)
- L1 cache with ECC protection
- iDMA engine for data movement
- FDS (Frequency/Droop Sensor) for power management
- Cluster control CSR registers

---

## 2. Position in Grid

The Overlay is distributed across the 4×5 mesh. DM complexes are associated with Tensix tiles and dispatch engines.

## 5. Feature Details

### 5.1 CPU Cluster

- DMCoresPerCluster = 8 RISC-V cores per complex
- NumDmComplexes = 14 total complexes
- Per-core reset: i_dm_core_reset_n[13:0][7:0]
- Per-complex uncore reset: i_dm_uncore_reset_n[13:0]

### 5.2 L1 Cache

- Multi-bank architecture
- ECC protection via EDC daisy-chain
- tt_t6_l1_wrap2, tt_t6_l1_pipe, tt_t6_l1_superarb, tt_t6_l1_latch_reg_array
- tt_t6_l1_rr_arb_tree: Round-robin arbitration (clk_i domain)

### 5.3 iDMA Engine

Protocol support (idma_pkg::protocol_e):
- AXI (0)
- OBI (1)
- AXILITE (2)
- TILELINK (3)
- INIT (4)
- AXI_STREAM (5)

Key modules:
- idma_transport_layer_rw_axi: AXI transport layer (clk_i domain)
- tt_idma_transport_layer_r_init_rw_obi: OBI transport layer

### 5.4 FDS (Frequency/Droop Sensor)
