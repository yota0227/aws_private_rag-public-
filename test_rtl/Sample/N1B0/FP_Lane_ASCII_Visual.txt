    ┌────────────────────────────────┐
    │  FP-LANE r1:                   │
    │  A[8..15]: 16-bit × 8 datums  │
    │  B[8..15]: 16-bit × 8 datums  │
    │                                │
    │  Booth: 8 pairs × 16×16 mult  │
    │  Result: 8 INT16 MACs/cycle   │
    └────────────────────────────────┘

    Total: 16 INT16 MACs/cycle per column


    INT8 MODE (2-PHASE TRANSPARENCY):
    ──────────────────────────────────

    Cycle N - Negative Phase (Latch Transparent):
    ┌────────────────────────────────────────────┐
    │  FP-LANE r0:                               │
    │  SRCA[0..7]: 16-bit datums (2 INT8 each)  │
    │  = 16 INT8 values                         │
    │                                            │
    │  Booth: 8 pairs × (8-bit×8-bit) × 2      │
    │  Result: 16 INT8 MACs (from Phase 1)     │
    └────────────────────────────────────────────┘

    Cycle N - Positive Phase (Latch Opaque):
    ┌────────────────────────────────────────────┐
    │  FP-LANE r1:                               │
    │  SRCA[0..7]: same datums (latched phase) │
    │  = 16 different INT8 values               │
    │                                            │
    │  Booth: 8 pairs × (8-bit×8-bit) × 2      │
    │  Result: 16 INT8 MACs (from Phase 2)     │
    └────────────────────────────────────────────┘

    Total: 32 INT8 MACs/cycle per column
    (Both phases in SAME cycle via ICG gating!)

KEY FACTS:

✓ FP-Lane is a COMPLETE multiply-accumulate unit
  └─ All 8 pipeline stages in one module
  └─ ~25,000 gates per lane
  └─ 256 lanes per Tensix × 25K = 6.4M gates

✓ Format-agnostic (same Booth hardware)
  └─ INT8/16/32, FP16B, FP32, TF32, MXFP4
  └─ Just reinterpret the bits

✓ Two-phase operation for INT8_2x
  └─ r0 and r1 lanes process different phases in same cycle
  └─ Doubles INT8 throughput (16 MACs instead of 8)

✓ Pipelined for sustained throughput
  └─ 8-cycle latency but 1 result/cycle
  └─ All 256 lanes active every cycle

✓ Zero-latency DEST integration
  └─ Latch array allows combinational read-modify-write
  └─ No stall cycles for back-to-back MACs
