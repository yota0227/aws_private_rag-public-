IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. **RTL-Verified Only**: All specifications derived from actual RTL instantiation
   counts and parameter values, NOT theoretical assumptions or official table
   reconciliation.

2. **FP-Lane = Individual Instantiation**: Each "FP-Lane" refers to one
   `tt_fp_lane` module instance, containing 8 MULT_PAIRS.

3. **Pipeline Depth = 5-6 Cycles**: RTL explicitly states 5 cycles
   (tt_fp_lane.sv:136), not 8.

4. **INT8_2x = Two-Cycle Instruction**: Not simultaneous dual-phase execution,
   but sequential phases with address offsetting and tag reuse via FF enable
   control.

5. **Throughput Pipelined**: Results assume continuous instruction stream.
   Actual throughput depends on instruction issue rate, dependencies, and
   register file access patterns.

6. **No Official Table Reconciliation**: The 2,048 INT16 value in official MAC
   table is NOT reconciled here. RTL-based calculation yields 1,024 INT16 per
   RTL cluster (4 Tensix) or 12,288 for full SoC. Discrepancy source unknown
   without further investigation.
