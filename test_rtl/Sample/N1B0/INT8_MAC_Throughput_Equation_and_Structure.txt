PART 1: INT16 MAC THROUGHPUT EQUATION
═══════════════════════════════════════════════════════════════════════════════

INT16 (BASE CASE - Single-Phase Latch Operation):
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ INT16_MACs_per_cluster = N_tensix × N_fp_lanes_per_tensix × INT16_MACs_lane│
│                                                                              │
│ Where:                                                                       │
│   N_tensix = 8 Tensix tiles per cluster                                   │
│   N_fp_lanes_per_tensix = 256 FP-Lanes per Tensix tile                    │
│   INT16_MACs_lane = 1 INT16 MAC per FP-Lane per cycle (single-phase)      │
│                                                                              │
│ INT16_MACs_per_cluster = 8 × 256 × 1 = 2,048 MACs/cycle                  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

HIERARCHICAL BREAKDOWN (INT16 specific):
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ Level 1: Single FP-Lane                                                     │
│ ──────────────────────                                                      │
│   INT16_MACs_per_lane = 1 MAC/cycle                                        │
│                                                                              │
│   Composition:                                                              │
│   • Single latch phase per cycle                                           │
│   • Processes 8 INT16 pairs (16 INT16 values from SRCA/SRCB)              │
│   • Produces 1 INT16 result (accumulate into DEST)                         │
│   • Total per cycle: 1 INT16 MAC/cycle                                     │
│                                                                              │
│ Level 2: M-Tile (2 FP-Lanes × 2 rows = 4 lanes)                          │
│ ──────────────────────────────────────────────────                         │
│   INT16_MACs_per_mtile = 4 lanes × 1 MAC/lane = 4 INT16 MACs/cycle       │
│                                                                              │
│ Level 3: G-Tile (2 M-Tiles = 8 lanes total)                               │
│ ──────────────────────────────────────────────                             │
│   INT16_MACs_per_gtile = 8 lanes × 1 MAC/lane = 8 INT16 MACs/cycle       │
│                                                                              │
│ Level 4: Tensix Tile (2 G-Tiles = 256 lanes)                              │
│ ──────────────────────────────────────────────                             │
│   INT16_MACs_per_tensix = 256 lanes × 1 MAC/lane = 256 INT16 MACs/cy    │
│                                                                              │
│ Level 5: Cluster (8 Tensix tiles = 2,048 lanes)                           │
│ ───────────────────────────────────────────────                            │
│   INT16_MACs_per_cluster = 256 lanes × 8 tensix × 1 MAC/lane            │
│   INT16_MACs_per_cluster = 2,048 lanes × 1 MAC/lane = 2,048 MACs/cycle  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

COMPARATIVE ANALYSIS (INT8 vs INT16):
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ INT16 Throughput per Cluster:                                              │
│ ─────────────────────────────────                                          │
│   INT16_MACs = 8 × 256 × 1 = 2,048 MACs/cycle                            │
│                                                                              │
│ INT8 Throughput per Cluster:                                               │
│ ────────────────────────────────                                           │
│   INT8_MACs = 8 × 256 × 4 = 8,192 MACs/cycle                             │
│                                                                              │
│ Multiplier Factor:                                                          │
│ ──────────────────                                                          │
│   INT8_MACs / INT16_MACs = 8,192 / 2,048 = 4×                            │
│                                                                              │
│ This 4× factor comes from:                                                  │
│   • 2× from two-phase latch transparency (negative + positive latch phases) │
│   • 2× from INT8 data packing (2 INT8 values per 16-bit slot)             │
│   • Combined: 2 × 2 = 4× throughput multiplier                            │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

DIAGRAM 3: INT8 vs INT16 THROUGHPUT MULTIPLIER DERIVATION
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ INT16 Baseline:                                                             │
│ ───────────────                                                             │
│   INT16_MACs_per_lane = 1 MAC/cycle                                        │
│   INT16_MACs_per_cluster = 256 lanes/tensix × 8 tensix × 1 = 2,048 MACs  │
│                                                                              │
│ INT8 Enhancement Factor Decomposition:                                      │
│ ───────────────────────────────────                                        │
│                                                                              │
│   Factor 1: Data Packing (2 INT8 per 16-bit slot)                         │
│   ──────────────────────────────────────────────                           │
│     • 16-bit Booth multiplier processes: A[15:0] × B[15:0]               │
│     • In INT8 mode, reinterpret as: A[7:0]×B[7:0] AND A[15:8]×B[15:8]  │
│     • This gives 2 parallel INT8 products from 1 Booth multiply           │
│     • Packing multiplier: 2×                                              │
│                                                                              │
│   Factor 2: Two-Phase Latch Transparency                                   │
│   ─────────────────────────────────────────                               │
│     • Negative latch phase: Captures and processes data                    │
│     • Positive latch phase: Independently captures different data          │
│     • Same FP-Lane hardware executes both phases in 1 cycle                │
│     • Two-phase multiplier: 2×                                            │
│                                                                              │
│   Combined INT8 Factor:                                                    │
│   ─────────────────────                                                    │
│     INT8_MACs_per_lane = INT16_MACs_per_lane × (Packing) × (Two-phase)   │
│     INT8_MACs_per_lane = 1 × 2 × 2 = 4 INT8 MACs/cycle                  │
│                                                                              │
│   Cluster-Level Impact:                                                    │
│   ────────────────────                                                     │
│     INT8_MACs_per_cluster = INT16_MACs_per_cluster × 4                   │
│     INT8_MACs_per_cluster = 2,048 × 4 = 8,192 INT8 MACs/cycle          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐

│  FP-LANE (Single Unit, Dual-Phase Capable)                                 │
│  ════════════════════════════════════════════════════════════════════       │
│                                                                              │
│  SRCA Latch Bank (48 rows × 16 datums × 2 banks)                          │

│  ─────────────────────────────────────────────                             │
│    [Row 0] → [B0, B1, B2, ..., B15] (16 INT16 values)                     │
│    [Row 1] → [B0, B1, B2, ..., B15]                                        │
│    ...                                                                      │

│  │                                                                    │   │
│  │ Selected datums: SRCA[even] × SRCB[even]                         │   │
│  │ Input: 8 INT16 pairs or 16 INT8 pairs (packed)                   │   │
│  │                                                                    │   │

│  │         ↓                                                          │   │

│  │         ↓                                                          │   │
│  │ ┌────────────────────────────────────────────────────────┐        │   │

│  │ │ INT8:  Add to DEST (2 accumulates, parallel)          │        │   │
│  │ └────────────────────────────────────────────────────────┘        │   │
│  │         ↓                                                          │   │
│  │ DEST Write: 2 INT8 results (phase 0)                              │   │

PART 3: FP-LANE DETAILED ARCHITECTURE DESCRIPTION
═══════════════════════════════════════════════════════════════════════════════

OVERVIEW: FP-LANE AS THE ATOMIC EXECUTION UNIT
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ An FP-Lane is a complete floating-point and integer multiply-accumulate     │
│ datapath that executes one (or more in INT8 mode) MAC operations per cycle. │
│                                                                              │
│ Key Characteristics:                                                        │
│ ──────────────────────                                                      │
│ • Single FP-Lane processes NUM_PAIR=8 operand pairs per cycle              │
│ • Supports all 14 numeric formats: INT8, INT16, INT32, FP32, FP16B, etc.  │
│ • 8-stage pipeline with 1-cycle throughput (fully pipelined)               │
│ • 8-cycle latency from input to DEST write                                 │
│ • 256 FP-Lanes per Tensix tile working in parallel                         │
│ • Two-phase latch transparency enables INT8 2× throughput                  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

DETAILED COMPONENT BREAKDOWN:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ Component 1: BOOTH MULTIPLIER (Stage S0)                                    │
│ ─────────────────────────────────────────                                   │
│   Function: Multiply two 16-bit (or packed 2×8-bit INT8) operands         │
│   Input:    A[15:0], B[15:0] (16-bit slots containing INT16 or 2×INT8)    │
│   Output:   8 partial products (one per Booth pair)                        │
│   Hardware: ~5,000 gates per lane                                          │
│   Format Agnostic: Works identically for INT8, INT16, FP16B, FP32         │
│                                                                              │
│   INT16 Mode:  Processes A[15:0] × B[15:0] → 8 products                   │
│   INT8 Mode:   Processes [A[7:0]×B[7:0]] AND [A[15:8]×B[15:8]]           │
│                 → 16 products (8 per sub-width), parallel                  │
│   Latency:     1 cycle (combinational)                                     │
│                                                                              │
│ Component 2: SOP COMPRESSOR / WALLACE TREE (Stage S1)                      │
│ ───────────────────────────────────────────────────────                    │
│   Function: Compress multiple partial products into single result          │
│   Input:    8 partial products from Booth multiplier                       │
│   Output:   22-bit sum-of-products value (INT16) or dual outputs (INT8)   │
│   Hardware: ~10,000 gates per lane (4:2 compressor cascade)                │
│   Technology: 4:2 compressor cells in tree topology                        │
│   Latency:     1 cycle                                                     │
│                                                                              │
│   INT16 Processing:                                                        │
│   └─ P0 + P1 + P2 + P3 + P4 + P5 + P6 + P7 → 22-bit result               │
│                                                                              │
│   INT8 Processing (dual-path):                                             │
│   ├─ Path A: P0 + P1 + P2 + P3 → 11-bit result (INT8_0)                  │
│   └─ Path B: P4 + P5 + P6 + P7 → 11-bit result (INT8_1)                  │
│                                                                              │
│ Component 3: EXPONENT PATH (Stage S2) [FP formats only]                    │
│ ────────────────────────────────────────────────────────                   │
│   Function: Calculate result exponent (parallel to multiply)                │
│   Input:    Exponent_A (5 or 8 bits), Exponent_B (5 or 8 bits)            │
│   Operation: Exponent_A + Exponent_B - 127 (bias correction)               │
│   Output:    Result exponent (adjusted)                                    │
│   Hardware: Simple adder (~500 gates)                                      │
│   Latency:  1 cycle (combinational)                                        │
│   Notes:    Integer formats (INT8/16/32) have implicit exponent = 0        │
│                                                                              │
│ Component 4: ALIGNMENT SHIFTER (Stage S3) [FP formats only]                │
│ ──────────────────────────────────────────────────────────────             │
│   Function: Align two operands to same exponent for FP addition            │
│   Input:    Multiplier result, Mantissa_B, Exponent_difference             │
│   Operation: Barrel shift B's mantissa by exponent difference              │
│   Output:    Aligned B value (same exponent as multiply result)            │
│   Hardware: Barrel shifter (~4,000 gates)                                  │
│   Shift Range: 0-64 bits (selectable in 1-bit increments)                 │
│   Latency:  1 cycle (combinational)                                        │
│   INT8/16 Path: Shifter bypassed (direct to adder)                         │
│                                                                              │
│ Component 5: CARRY-LOOKAHEAD ADDER (Stage S4)                              │
│ ──────────────────────────────────────────────                             │
│   Function: Add aligned/shifted values + DEST accumulator                  │
│   Input:    SOP result (22-bit), Aligned_B (22-bit), DEST (32-bit)        │
│   Operation: SOP + DEST + (aligned_B for FP, or 0 for INT)                │
│   Output:    Sum result (32-bit)                                           │
│   Hardware: CLA adder (~3,000 gates)                                       │
│   Carry Path: Looks ahead 4 bits at a time                                │
│   Latency:  1 cycle (combinational)                                        │
│                                                                              │
│ Component 6: LEADING ZERO COUNTER & NORMALIZER (Stages S5-S6)              │
│ ────────────────────────────────────────────────────────────────            │
│   Function: Normalize result to proper FP format (find MSB position)        │
│   Input:    32-bit sum from CLA adder                                      │
│   Process Step 1 (LZC): Count leading zeros → shift amount                │
│   Process Step 2 (Normalize): Left-shift to place hidden bit at MSB        │
│   Output:   Normalized mantissa + adjusted exponent                        │
│   Hardware: LZC (~2,000 gates) + Shifter (~3,000 gates)                   │
│   Latency:  2 cycles (S5 and S6)                                           │
│   INT8/16 Path: Simplified normalization (fewer stages)                    │
│                                                                              │
│ Component 7: ROUNDER (Stage S6, overlapped with normalization)             │
│ ──────────────────────────────────────────────────────────────              │
│   Function: Apply rounding based on Guard/Round/Sticky bits                │
│   Input:    Normalized mantissa + exponent + G/R/S bits                    │
│   Rounding Modes: RNE (round-to-nearest-even), RTZ, RDN, RUP, RZU         │
│   Output:   Rounded value (may cause exponent adjustment if overflow)      │
│   Hardware: Mux tree + ripple carry (~2,000 gates)                         │
│   Latency:  1 cycle (part of S6)                                           │
│                                                                              │
│ Component 8: OUTPUT REGISTER / DEST WRITE PORT (Stage S7)                  │
│ ───────────────────────────────────────────────────────────────             │
│   Function: Register final result and drive DEST write signal              │
│   Input:    Rounded result (32-bit)                                        │
│   Storage:  Internal output register (captures result)                     │
│   Output:   Valid flag + result + DEST address                            │
│   Hardware: 32-bit register + write control logic                          │
│   Latency:  1 cycle (sampling at end of pipeline)                          │
│   Capability: Can write to DEST 0-latency combinational (latch-based DEST) │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

REGISTER FILE INTEGRATION (Inputs & Outputs):
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ SRCA (Source A Register Bank):                                              │
│ ──────────────────────────────                                              │
│   Capacity:     48 rows × 16 datums × 2 banks = 768 INT16 entries/bank    │
│   Width:        16 bits per datum (INT16 or 2×INT8 or other formats)       │
│   Access:       Dual-bank, 0-latency latch array (combinational read)      │
│   Purpose:      Provides operand A to FP-Lane inputs                       │
│   Refresh:      Bank switches during execution for double-buffering        │
│                                                                              │
│ SRCB (Source B Register Bank):                                              │
│ ──────────────────────────────                                              │
│   Capacity:     64 rows × 16 datums × 2 banks = 1,024 INT16 entries/bank  │
│   Width:        16 bits per datum                                          │
│   Access:       Dual-bank, 0-latency latch array (combinational read)      │
│   Purpose:      Provides operand B to FP-Lane inputs                       │
│   Refresh:      Bank switches during execution for double-buffering        │
│                                                                              │
│ DEST (Destination Register File):                                           │
│ ─────────────────────────────────────────                                   │
│   Capacity:     512 rows × 4 columns × 2 banks = 2,048 INT32 entries/bank │
│                 (or 4,096 INT16 entries if repacked)                       │
│   Width:        32 bits per entry (can hold INT32, FP32, 2×INT16, 4×INT8) │
│   Access:       Zero-latency combinational read-modify-write (latch-based) │
│   Purpose:      Accumulator for MAC results (writes result each cycle)     │
│   Banking:      Dual-bank prevents read-write conflicts                    │
│   Performance:  Enables back-to-back MACs without stall                    │
│                                                                              │
│ LATENCY HIDING IN PIPELINE:                                                │
│ ─────────────────────────────                                              │
│   DEST writes occur at Stage 7 (latency = 8 cycles)                       │
│   DEST reads happen at Stage 4 (from previous MAC, 4 cycles prior)        │
│   No structural hazard because reads and writes are pipelined              │
│   Result: Continuous stream of MACs with no pipeline stalls                │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

KEY ARCHITECTURAL INSIGHTS
═══════════════════════════════════════════════════════════════════════════════

1. TWO-PHASE LATCHES ARE THE ENABLER
   ─────────────────────────────────
   • Negative/Positive latch phases capture independently in same cycle
   • No performance penalty—same hardware executes both phases
   • Enables "2× throughput" without 2× power (pipelined throughput)

2. INT8 DATA PACKING EXPLOITS BOOTH MULTIPLIER WIDTH
   ──────────────────────────────────────────────────
   • 16-bit Booth multiplier naturally handles 2×INT8
   • No additional multiplier hardware needed
   • Compression/decompression in control path only

3. THE 4× MULTIPLIER IS STRICTLY CORRECT
   ──────────────────────────────────────
   • Official table: INT8 = 8,192 MACs vs INT16 = 2,048 MACs
   • 8,192 / 2,048 = 4.0 exactly
   • Derives from: 2× (two-phase) × 2× (INT8 packing)

4. DEST REGISTER FILE MUST SUPPORT DUAL WRITE
   ──────────────────────────────────────────
   • Each phase produces independent results
   • Two DEST entries written per cycle
   • Resolved through port arbitration and banking

5. RTLVERIFIED PARAMETERS
   ─────────────────────
   • ENABLE_INT8_PACKING = 1 (tt_t6_proj_params_pkg.sv:16)
   • HALF_FP_BW = 1 (enables two-phase operation)
   • INT8_2x format gated separately from UINT8_2x (RTL controlled)
