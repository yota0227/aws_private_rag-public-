## LEVEL 6: INT8 vs INT16 PHASE DIFFERENCES

Both lanes r0/r1 execute in SAME cycle (via ICG two-phase transparency):

INT16 Mode:
┌──────────────────────────────┐
│ Cycle N                      │
│                              │
│ r0: Load INT16[0:7]          │
│     Multiply 8 pairs         │
│     → 8 INT16 MACs           │
│                              │
│ r1: Load INT16[8:15]         │
│     Multiply 8 pairs         │
│     → 8 INT16 MACs           │
│                              │
│ Total: 16 INT16 MACs/cycle   │
└──────────────────────────────┘

INT8 Mode (2-phase within same cycle):
┌───────────────────────────────────────┐
│ Cycle N - Negative phase              │
│ (Latch transparent)                   │
│                                       │
│ r0: Load INT8[0:15] (16 values)       │
│     From: 8 datums × 2 INT8/datum     │
│     Multiply 8 pairs                  │
│     → 16 INT8 MACs (2 per pair)       │
│                                       │
│ Cycle N - Positive phase              │
│ (Latch opaque)                        │
│                                       │
│ r1: Load INT8[16:31] (16 values)      │
│     From: same 8 datums, different    │
│           phase capture               │
│     Multiply 8 pairs                  │
│     → 16 INT8 MACs (2 per pair)       │
│                                       │

## LEVEL 9: SIGNAL FLOW EXAMPLE (INT16 MAC)

Input: A[i] (sign, exp, mantissa), B[j] (sign, exp, mantissa), DEST[k]

S0 (Booth):
   Booth_prod[0:7] = A.mantissa × B.mantissa (8 products, 32-bit each)

S1 (SOP):
   SOP_man = Σ(Booth_prod[0:7]) = 22-bit accumulated mantissa

S2 (Exp path + Align):
   exp_prod = A.exp + B.exp - 127 (bias)
   shift_amt = exp_prod[4:0]  (alignment amount)
   B_aligned = B.mantissa >> shift_amt

S3 (CLA add):
   acc = B_aligned + SOP_man + DEST[k]
   Result: 32-bit accumulation value

S4-S5 (Normalize + Round):
   lz_count = leading_zeros(acc)
   acc_normalized = acc << lz_count
   exp_final = exp_prod - lz_count + rounding_exp_adjust
   rounded_mantissa = round(acc_normalized, rounding_mode)

S6 (Format convert):
   If INT16 format:
      Extract MSB 16 bits as INT16 result
   Else if FP16B:
      Extract sign, 5-bit exp, 10-bit mantissa

S7 (Register):
   o_result = format_result
   o_result_valid = 1

Output: DEST[k] written with result

Time: 8 cycles from input to DEST write

## LEVEL 10: DESIGN HIGHLIGHTS

1. FORMAT-AGNOSTIC BOOTH MULTIPLIER
   └─ Same hardware multiplies INT8, INT16, FP16B, FP32, TF32
      No format-specific multiplier branches needed

2. PIPELINED SOP COMPRESSOR
   └─ Wallace tree compresses 8 partial products in 1 cycle
   └─ Maintains 22-bit precision for extended accumulation

3. DUAL EXPONENT+MANTISSA PATHS (Parallel)
   └─ Exponent processing (exp_A + exp_B - bias) in parallel
   └─ Mantissa processing (alignment + addition) in parallel
   └─ Both finish at same time for final normalization

4. ZERO-LATENCY DEST INTEGRATION
   └─ CLA adder reads DEST in same cycle it accumulates
   └─ Latch array (not SRAM) provides combinational access
   └─ No stall cycles for back-to-back MACs

5. TWO-PHASE LATCH TRANSPARENCY (INT8_2x mode)
   └─ ICG gating enables two independent phases in 1 cycle
   └─ r0 and r1 lanes process 2× data without doubling hardware
   └─ Effective 16 INT8 MACs/cycle instead of 8

6. FULL PIPELINE UTILIZATION
   └─ 8-stage pipeline always active (no conditional stalls)
   └─ K-loop iteration doesn't stall pipeline (refill happens during execution)
   └─ Achieves 1 result/cycle in steady state
