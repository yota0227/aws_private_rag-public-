                        12-CLUSTER SYSTEM OVERVIEW
═══════════════════════════════════════════════════════════════════════════════════════

A 12-cluster N1B0 system = 12 × (1 Cluster with 8 Tensix tiles each)

                          ┌──────────────────┐
                          │  N1B0 SoC with   │
                          │  12 Clusters     │
                          │                  │
                          │  96 Tensix tiles │
                          │  (12×8)          │
                          │                  │
                          └──────────────────┘


═══════════════════════════════════════════════════════════════════════════════════════
                    DETAILED CLUSTER BREAKDOWN (1 Cluster)
═══════════════════════════════════════════════════════════════════════════════════════

Per Single Cluster:
├─ 8 Tensix tiles
├─ 16 G-Tiles (2 per Tensix)
├─ 32 M-Tiles (2 per G-Tile)
├─ 256 FP-Tiles (8 per M-Tile)
└─ 512 FP-Lanes (2 per FP-Tile)

Performance per Single Cluster @ 1.5 GHz:
├─ INT16: 32,768 MACs/cycle × 1.5 GHz = 49,152 GOPS = 49.152 TOPS
└─ INT8:  65,536 MACs/cycle × 1.5 GHz = 98,304 GOPS = 98.304 TOPS


═══════════════════════════════════════════════════════════════════════════════════════
                    12-CLUSTER SYSTEM CALCULATION
═══════════════════════════════════════════════════════════════════════════════════════


STEP 1: Single Cluster Performance
──────────────────────────────────

Per Cluster (1 cluster = 8 Tensix):

  INT16:  32,768 MACs/cycle × 1.5 GHz = 49.152 TOPS
  INT8:   65,536 MACs/cycle × 1.5 GHz = 98.304 TOPS


STEP 2: Scale to 12 Clusters
────────────────────────────

INT16 Performance:
├─ 12 clusters × 32,768 MACs/cluster/cycle = 393,216 MACs/cycle
├─ 393,216 MACs/cycle × 1.5 GHz = 589,824 GOPS
└─ 589,824 GOPS = 589.824 TFLOPS (589.824 TOPS)

INT8 Performance (2-phase):
├─ 12 clusters × 65,536 MACs/cluster/cycle = 786,432 MACs/cycle
├─ 786,432 MACs/cycle × 1.5 GHz = 1,179,648 GOPS
└─ 1,179,648 GOPS = 1,179.648 TFLOPS (1,179.648 TOPS)

INT16 Calculation:
──────────────────

Total MACs/cycle = 12 clusters × 32,768 MACs/cluster/cycle
                 = 393,216 MACs/cycle

TOPS = (393,216 MACs/cycle × 1.5 GHz) / 1,000
     = 589,824 GOPS / 1,000
     = 589.824 TFLOPS


INT8 Calculation (2-phase):
───────────────────────────

Total MACs/cycle = 12 clusters × 65,536 MACs/cluster/cycle
                 = 786,432 MACs/cycle

TOPS = (786,432 MACs/cycle × 1.5 GHz) / 1,000
     = 1,179,648 GOPS / 1,000
     = 1,179.648 TFLOPS

═══════════════════════════════════════════════════════════════════════════════════════
                    FORMULA FOR N-CLUSTER SYSTEMS
═══════════════════════════════════════════════════════════════════════════════════════

For any N-cluster system @ frequency F GHz:

INT16 TOPS = (N × 32,768 MACs/cluster/cycle × F GHz) / 1,000

INT8 TOPS  = (N × 65,536 MACs/cluster/cycle × F GHz) / 1,000


Examples:
─────────

For N=12, F=1.5 GHz:
├─ INT16: (12 × 32,768 × 1.5) / 1,000 = 589.824 TOPS ✓
└─ INT8:  (12 × 65,536 × 1.5) / 1,000 = 1,179.648 TOPS ✓

For N=8, F=1.5 GHz:
├─ INT16: (8 × 32,768 × 1.5) / 1,000 = 393.216 TOPS ✓
└─ INT8:  (8 × 65,536 × 1.5) / 1,000 = 786.432 TOPS ✓

For N=16, F=2.0 GHz:
├─ INT16: (16 × 32,768 × 2.0) / 1,000 = 1,048.576 TOPS
└─ INT8:  (16 × 65,536 × 2.0) / 1,000 = 2,097.152 TOPS

                              KEY INSIGHTS
═══════════════════════════════════════════════════════════════════════════════════════

✓ 12-CLUSTER SYSTEM ACHIEVES:
  ├─ 589.824 TFLOPS @ 1.5 GHz (INT16)
  └─ 1,179.648 TFLOPS @ 1.5 GHz (INT8)

✓ PERFECT LINEAR SCALING:
  ├─ Each cluster independently delivers 49.152 TOPS (INT16) / 98.304 TOPS (INT8)
  ├─ 12 clusters = 12× single-cluster performance
  └─ No architectural bottleneck or contention

✓ INT8 2-PHASE THROUGHPUT:
  ├─ Latch transparency enables 2× INT8 MACs vs INT16
  ├─ Same hardware, different operand packing
  └─ 1,179.648 TFLOPS = 2.0× 589.824 TFLOPS ✓

✓ FREQUENCY SCALING:
  ├─ Performance is linear with clock frequency
  ├─ At 2.0 GHz: 786.4 TFLOPS (INT16) = 1.33× baseline
  ├─ At 3.0 GHz: 1,179.6 TFLOPS (INT16) = 2.0× baseline
  └─ Can extrapolate for any frequency

✓ MEMORY SCALABILITY:
  ├─ 9.216 MB L1 cache (distributed across 96 Tensix tiles)
  ├─ No contention (tile-local L1 design)
  └─ Sufficient for K=8192 GEMM operations per tile
